前言

ConcurrentHashMap 是 HashMap 的线程安全版本，本文将对 Java8 的 ConcurrentHashMap 进行分析。

ConcurrentHashMap 和 HashMap 类似，底层都是使用哈希桶来保存结点，当某一个桶的节点数大于 8 时，链表会转化为红黑树，这样子会提高查询速度。链表的时间复杂度为 O(n)，而红黑树的时间复杂度为 O(logN)。

哈希桶初始化过程

初始化过程中使用 while 循环判断每次 tab 是否为空，这样当多个线程同时操作 ConcorrentHashMap 的时候，每次都会判断一次哈希桶是否为空。

这里 sizeCtl 是一个用于同步多个线程的共享变量，如果它的当前值为负数，则说明哈希桶正在被某个线程初始化或者扩容，所以，如果某个线程想要初始化哈希桶或者对哈希桶扩容，需要去竞争 sizeCtl 这个共享变量，获得变量的线程才有许可去进行接下来的操作，没能获得的线程将会一直自旋来尝试获得这个共享变量，所以获得 sizeCtl 这个变量的线程在完成工作之后需要设置回来，使得其他的线程可以走出自旋进行接下来的操作。

在 initTable 方法中我们可以看到，当线程发现 sizeCtl 小于 0 的时候，他就会让出 CPU 时间，而稍后再进行尝试，当发现 sizeCtl 不在小于 0 的时候，就会通过调用方法 compareAndSwapInt 来讲 sizeCtl 共享变量变为 -1，以告诉其他试图获得 sizeCtl 变量的线程，目前正在由本线程在享用该变量，在我完成我的任务之前你可以先休息一会，灯虎再来试试吧，我完成工作之后会释放掉的。

而其他线程在发现 sizeCtl 小于 0 的时候就会理解这种交流，他们会让出 cpu 时间，等待下次调度再来尝试 sizeCtl 来进行自己的工作。在完成初始化 table 的任务之后，线程需要将 sizeCtl 设置为其他线程变量获取变量的状态，这其中还有一个地方需要注意，就是在某个线程通过 U.compareAndSwapInt 方法设置了 sizeCtl 之前和之后进行了两次 check，来检测 table 是否被初始化过了，这种检测是必须的，因为在并发环境下，可能前一个线程正在初始化 table 但是还没有成功初始化，也就是 table 依旧为 null，而有一个线程发现 table 为 null，他就会竞争 sizeCtl 来进行 table 初始化，但是当前线程在完成初始化之后，那个试图初始化 table 的线程获得了 sizeCtl，但是此时 table 已经被初始化了，所以，如果没有再次判断的话，可能会就将之后进行 put 操作的线程的更新覆盖掉，这是极为不安全的行为。

获取元素值过程

和 HashMap 一样，ConcurrentHashMap 底层也是维护的也是数组+链表，在链表长度大于 8 转化为数组+红黑树。

首先通过计算 key 的哈希值，再通过对数组大小取模来得到元素值存放的下标索引。然后判断该位置上是否为 null，如果为 null，则返回 null。否则，如果该位置上的第一个元素（链表头结点或者红黑树的根节点）与我们先要查找的记录匹配，则直接返回该节点的值，否则，需要遍历链表或者红黑树。

但是 ConcurrentHashMap 是一种线程安全的 HashMap，但是却没有在 get 方法上加锁来进行同步，因为 Java8 中对 Node 做了一些优化。

我们发现哈希桶是被 volatile 关键字修饰的，这就代表我们不需要担心 table 数组的线程可见性问题，也就没有必要再加锁来实现并发了。

插入键值对过程

在进行 put 操作的时候，我们可能会发现 table 数组还没有初始化的情况，或者发现 table 中容纳的记录数量超过了阈值的情况，前者我们需要进行 table 的初始化，而后者我们需要对 table 进行扩容操作。初始化已经讲过了，下面讲讲扩容吧。第一，我们需要计算这个 key 的哈希值，并且对其进行取模运算得到它在数组存放的 index，然后将其放在对应位置里面的链表或者红黑树中去，如果其中要存放的链表长度超过 8，需要转换为红黑树。如果超过了可存储容量，则需要将其进行扩容。

首先，计算 key 的哈希值，然后计算 key 存放的 index，然后获取哈希同种 index 位置上的值，如果为 null，说明该位置上还没有记录，则通过 casTabAt 方法来将键值对插入到 table 的index 位置上，否则，通过 synchronized 关键字对哈希桶的 index 位置加锁，需要注意的是，当前线程只会锁住哈希桶的 index 位置，其他位置上没有锁住，所以此时其他线程可以安全的获得哈希桶上的其他位置来进行操作。这就提高了 ConcurrentHashMap 的并发度。
