自我介绍。

略

会不会 mongodb?

暂时不会。

会不会 memcached?

暂时不会。

集群环境怎么搭建的？

用户通过访问 nginx 代理服务器，nginx 将请求分发到 tomcat 集群中的某一台服务器上。

为什么要做集群？

  • 降低服务器压力。
  • 容灾。
  • 增大吞吐量。

反向代理的原理说下？

反向代理是指以代理服务器的方式来接受来自客户端的请求，然后将请求转发给内部网络上的服务器，并将从服务器上得到的返回结果返回给网络上客户端，这个时候代理服务器就表现为一个服务器。

负载均衡的原理说下？

和反向代理的原理差不多。

知道哪几种分流算法？

轮询，权重，ip_hash,url_hash,fair五种。

nginx 需要隔离出来单独做一台服务器吗？

我回答出来的是说隔离出来作为一台服务器，

如果 nginx 挂掉能不能访问 Tomcat ?

不能。

Session 共享怎么做的？

将 session 信息存储到 redis 中。

单点登录和单点注销原理说下？

将 session 信息存储到 redis 中，同时将 session_id 写回客户端，每次登陆判断请求中传过来的 session_id 是否存在于 redis 中，如果存在，则刷新该 session_id 的超时时间，如果不存在则重新登陆。单点注销类似。

除了用 cookie 做 session 追踪还会什么方法做 session 追踪？

不会啊。用 DNS ，你没用过就算了，下一个。

购物车的设计思路和数据库里怎么设计的？

用户 id 做外键来标识购物车。

分布式锁了解多少？setnx 是不是原子操作？那 getset 是不是原子操作？

都是原子操作。

mysql 用过吗？分组的关键字是什么？

group by，分组之后能不能再次选择？能啊，用什么关键字？ having 。

分组+选择条件还能再接 order by 进行排序吗？

能啊？确定能？确定啊？真的确定？真的确定？这里几个反问问的我心里发毛，现在我都不知道能不能在排序了。

对什么数据结构最了解？

hashmap啊。

那 hashset 了解吗？

不会原理，只会用。

那你说下 hashmap 的原理吧。

数组 + 链表。

为什么 Java7 中产生哈希碰撞之后要用头插法插到链表里面？

因为头插最快啊。

hashmap 怎么根据 key 来获取值。

将 key 进行哈希运算得到哈希码，将哈希码进行取模得到索引，得到索引之后通过遍历链表得到具体的 value。

hashmap 中用了哪种哈希算法？

高位运算好像。

那你说下 hashmap 和 currentHashmap 的区别吧？

currentHashmap分段锁，提高并发量，锁数量决定并发数。

那 hashtable 有用过吗？说下原理。

加了 synchronized 的hashmap。

产生死锁的四个条件是什么？

循环等待，不可剥夺，占有且等待，最后一个没想出来，还有一个互斥。

知道 arp 吗？

知道啊，IP 地址转化为 mac 地址。

那你说下 ARP 欺骗的原理吧？

不会，我只知道 ARP 欺骗是属于 web 攻击的一种，和 xss,ddos,sql 注入一起的。
