支付宝当面付怎么做的？

  1. 发送的消息：本地数据库已经生成支付记录，然后发送给支付宝的消息只需要包括订单编号即可。
  2. 网关：决定应用发送支付请求的地址。
  3. 私钥公钥：非对称加密，使用公钥加密，服务器使用私钥解密。
  4. 支付宝响应：这里会响应给自己一串 Json 数据，里面包含二维码生成链接。
  5. 后端生成二维码：使用 google 的开源框架生成。
  6. 用户扫描二维码：这里再次发起对支付宝的请求。
  7. 支付宝响应：支付宝并不知道我本机的 ip，为了需要让支付宝知道我的本机地址，我需要借助外在工具 natapp,花生壳，向外提供一条支付宝响应到本机的隧道。
  8. 更改订单状态：从支付宝响应中拿到 json 数据，解析 json，判断是否支付成功。成功则修改订单的状态变成已支付。

sleep() 和 wait() 理解

  1. sleep() 可以在任何地方使用，wait() 只能在同步方法以及同步块中使用。
  2. sleep() 方法是 Thread 的方法，调用之后会暂停此线程指定的时间，但是监控依然保持，不会释放对象锁，而是到时间自动恢复。wait() 方法是 Object 的方法，调用会放弃对象锁，进入等待队列，需要调用 notify 和 notifyAll 来唤醒，才会进入锁池，再次竞争锁才会进入运行状态。
  3. 调用 sleep() 方法则当前的线程会让出 cpu，但是并不会释放锁资源，然后 cpu 去执行其他任务，在 sleep 时间到了，自动往下执行。调用 wait() 则让当前线程让出同步锁，等到另外的线程 notify 或者 notifyAll 唤醒，才继续竞争同步锁。
  4. sleep() 调用会进入就绪状态，wait() 调用会进入阻塞状态。（这一行属于个人理解）
