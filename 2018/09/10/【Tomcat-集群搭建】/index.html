什么是 Tomcat 集群？

集群分为两种，一种是单机部署多应用，一种是多机部署多应用。

  • 单机部署多应用：一台服务器上运行多个 Tomcat，每个 Tomcat 中都运行了一个 Web 应用，需要解决端口冲突问题，每个端口必须是唯一的。比如说一个 Tomcat 开放的端口是 8080，另一个 Tomcat 开放的端口可以是 9080。
  • 多级部署多应用：可以有多台服务器，每个服务器上都有一个 Tomcat，每个 Tomcat 中都运行了一个 Web 应用，这种方式不需要解决端口冲突问题，但是需要解决防火墙端口开放问题，比如将 8080 端口开放出来。

集群之间通过何种方式进行连接？

不管是单机部署多应用还是多机部署多应用，都需要通过代理服务器来进行负载均衡的配置。这里可以使用 nginx 来进行配置。用户的请求如果不通过 nginx 服务器就会直接进入我们的服务器中，如果设置了 nginx 服务器，可以通过轮询或者随机的方式来进行多台服务器的访问，下面介绍如何搭建 Tomcat 集群。
