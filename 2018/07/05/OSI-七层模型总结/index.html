OSI 七层模型总结

2018-07-05

OSI 七层模型总共有哪七层？

  1. 物理层
  2. 数据链路层
  3. 网络层
  4. 传输层
  5. 会话层
  6. 表现层
  7. 应用层

OSI 七层模型每一层的作用是什么？

  1. 物理层：数据以二进制数据形式在物理媒体上传输。
  2. 数据链路层：基于物理层的比特流服务，相邻节点之间建立数据链路，通过差错控制提供数据帧在信道上无差错的传输，并进行各电路上的动作系列。数据链路层在不可靠的物理介质上提供可靠的传输。该层的作用是：物理地址寻址、数据封装成帧、流量控制、数据的检错、重发等等。
  3. 网络层：在两台通信的计算机之间可能会有多个数据链路，也有可能会经过多个通信子网。网络层的任务就是选择合适的网间路由和交换节点，确保数据及时发送。网络层将数据链路层提供的帧封装成数据包，包中封装有网络层包头，其中还有逻辑地址信息，比如说源站点和目的地址的网络地址。
  4. 传输层：传输层的数据单元也称作数据包，但是对于 TCP 而言称为段，对于 UDP 而言称为数据报。传输层负责获取全部信息，因此，它必须跟踪数据单元碎片、乱序到达的数据包和其它在传输过程中可能发生的危险。传输层为会话层提供端到端的透明的、可靠的数据传输服务，也就是说传输层对会话层屏蔽了通信传输系统的具体细节。
  5. 会话层：会话层不参与具体的传输，它包括访问验证和会话管理在内的建立和维护应用之间通信的机制。如服务器验证用户登录便是由会话层完成的。
  6. 表示层：表示层提供格式化的表示和转换数据服务。数据的压缩和解压缩，加密和解密等工作都由表示层负责。
  7. 应用层：应用层为操作系统或网络应用程序提供访问网络服务的接口。

OSI 七层模型每一层有哪些常见的协议？

  1. 物理层：IEEE802。
  2. 数据链路层：PPP，ARP。
  3. 网络层：IP，ICMP，RIP，OSPF，BGP，IGMP。
  4. 传输层：TCP，UDP。
  5. 会话层：无
  6. 表示层：无
  7. 应用层：HTTP，SNMP，FTP，SMTP，DNS，Telnet。
