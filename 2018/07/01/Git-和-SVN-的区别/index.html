Git 和 SVN 的区别

2018-07-01

  1. SVN 是集中式的，而 Git 是分布式的。集中式的版本控制是说版本库是放在一个中央服务器上面，而每次程序员进行开发的时候，都需要从中央服务器上面拉取最新的代码。而分布式的版本控制是说每个人的电脑上都是一个版本库，两个人同时修改了同一个文件的时候，只需要将各自的修改推送给对方就能让对方知道修改的内容了。基于这两个特点，集中式的版本库一旦出现问题，那么所有人将无法工作，而分布式的版本库当其中一个版本库出现问题，只需要在其他的版本库中复制一个即可。

  2. SVN 支持空目录，而 Git 不支持。Git 无法提交空目录到仓库中，SVN 可以。

  3. Git 可以在不联网的条件下查看所有的提交日志，而 SVN 是需要联网才行。

  4. Git 适合开源的项目进行开发，而 SVN 不适合。Git 可以每个人维护自己的版本，而 SVN 维护的是一个中心版本。

  5. SVN 有严格的权限控制，而 Git 没有。在公司的项目中，为了避免所有的人都看到项目的所有代码，可能会使用 SVN 管理开发人员的可查看权限。而 Git 因为克隆下来的版本库就是所有的版本，所以可以查看别人的代码。

  6. 分支的使用不一样。Git 是针对整个仓库进行分支的建立，而 SVN 可以针对某个目录来进行分支的建立。

  7. SVN 下载源代码慢，Git 下载速度快。
